# Agent-Workflow Template Pack — one-pager

> **Status:** `reference`

## What it is
A drop-in Markdown + shell "session discipline" scaffold: an agent constitution template, a session-log card template, a one-page discipline playbook, and three advisory Claude Code hooks that print and exit 0. No dependencies, no runtime, no enforcement — it is conventions and reminders you drop into a repo.

## Who it's for
Teams running Claude Code / agent sessions who want a conventions starter — a shared shape for how sessions are logged, how work is claimed before it's built, and what "good enough" means — without writing it from scratch.

## What's real today
All the templates and the fail-open hooks ship and work as text and shell. The parallel-safe claim-before-build convention is spelled out so multiple agents don't collide. Everything is plain files you can read and edit.

## What's NOT (honest v0.x)
These are conventions and reminders, not enforcement — the hooks are advisory and fail open (they print and exit 0). Nothing here blocks a commit or gates CI.
