**Title:** Agent Fleet Field Manual — Operating Lessons for Autonomous Coding-Agent Fleets

**Tagline:** The scar tissue of running an agent fleet, written down and cited to real artifacts — so your fleet stops manufacturing confident, unverified claims.

## Description (≈120 words)

Autonomous coding agents fail in one characteristic way: they overstate. They synthesize a payload from memory and call it a test. They mark work done before it is. They report "green in CI" when the check never ran the code. This field manual is eleven chapters of the mechanical disciplines that stop that — drawn from one real revenue lab and cited, line by line, to the commit, pull request, or file that earned each lesson. You get the D1 payment-verification lesson, the born-red work-tracking rule, the six-field money protocol that lets agents handle revenue without spending it, the merge-wall refusal discipline, and honest negative ledgering. Where the evidence is thin, the book says so. Two chapters are free. Includes three copy-paste templates.

## Feature bullets

- **Every lesson is cited** — each claim points at a real commit SHA, pull request, or file, so you can check it instead of trusting it.
- **The D1 payment lesson (free chapter)** — why 13 green tests lied, and how to test payment code against vendored real payloads at the HTTP layer.
- **Born-red work tracking** — a mechanical rule that makes it impossible for a unit of work to certify itself done by default.
- **The six-field owner-action grammar** — let agents produce revenue leverage while a human keeps the trigger; no agent spends, publishes, or creates accounts.
- **Merge-wall discipline** — deny wins, the first denial is terminal, never reword or tunnel; the child builds to green and a human lands it.
- **Honest negative ledgering** — a worked case study of a build that overran its budget ~2.3× and was recorded in the open, because a failure you wrote down is a deliverable.
- **Three runnable templates** — a born-red session card, a six-field owner-action block, and a kill-rule intake skeleton, ready to copy.
- **Reads anywhere, builds anywhere** — a single self-contained HTML file (no CDN, no fonts, no tracking) plus the raw markdown; rebuild with stdlib Python, zero dependencies.

## FAQ

**Q: Is this a tutorial for a specific framework?**
No. It is framework-agnostic operating discipline. The examples come from a Claude Code / SDK-harness lane, but the lessons — verify what a check runs, keep the money trigger human, track work so it can't self-certify — transfer to any agent fleet.

**Q: Are the case studies real?**
Yes, and that is the point. Every lesson cites the actual repository artifact that produced it — a commit, a pull request, a file, a verbatim classifier denial. Nothing is invented. Where the evidence is thin or a claim rests on an unverified statement, the book flags it in the text.

**Q: What do I actually get?**
A single self-contained HTML book, the eleven source markdown chapters, and three copy-paste templates, in one zip. Two chapters (the D1 lesson and the 13-green-tests trap) are free to read before you buy.

**Q: Is it going to date?**
Some of it will. The underlying tooling — a specific classifier, a specific branch-protection posture — changes, and the book says so plainly. You are buying the patterns and the honesty about their limits, not a spec frozen in time.
