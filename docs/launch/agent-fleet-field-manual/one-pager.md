## The eleven chapters (2 free)

- **00 — Preface.** Thesis (honesty over hype; every claim cited), audience, and a plain disclaimer that these are one lane's lessons.
- **01 — The D1 Lesson.** **FREE.** Never claim a payment path works without executing it; test against vendored real payloads at the HTTP layer.
- **02 — The 13 Green Tests Trap.** **FREE.** A green check that contradicts visible evidence is a bug in the check; require non-author real-path verification before any sell-click.
- **03 — Born-Red Session Cards.** Work opens red, flips complete last; the merge holds while red.
- **04 — The Owner-Action Grammar.** The six-field money interface; agents queue click-level instructions, never spend.
- **05 — Merge Walls and Classifier Denials.** Deny wins; first denial is terminal; child builds to green, coordinator lands.
- **06 — Single-Writer Control Files.** One writer per control file prevents lost-update races.
- **07 — The Pacemaker and the Failsafe.** Fast self-chaining pacemaker + slow cron failsafe; decide and flag, never idle undefined.
- **08 — Kill-Rule Intake Discipline.** Every candidate declares how it dies before it's built.
- **09 — Honest Negative Ledgering.** A measured, recorded failure is a deliverable.
- **10 — Appendix: Runnable Templates.** The three shipped templates, shown inline.

## What it is NOT (honest)

- Not a framework tutorial and not tied to one SDK — it is framework-agnostic operating discipline.
- Not universal law — one small lane's lessons; some figures are single observations; some claims rest on an owner statement that was not agent-side verified (flagged in the text, e.g. Chapter 5's branch-protection caveat).
- Not evergreen — the underlying tooling changes and some lessons will date; the book says so.
