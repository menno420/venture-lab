# Recovery checklist — <Book Title>

For any drafting run that died mid-flight (guide chapter 5). Work it
top to bottom; forward-only — never rewrite the failure's record.

## Before every run (prevention — one minute)

- [ ] A committed/synced note declares this run: book, band, chapter
      plan, marked **in-progress**.
- [ ] Commit (or save/export) at every chapter boundary.
- [ ] **Push/sync after every commit** — unpushed work does not exist
      for recovery purposes.

## When a run dies

- [ ] **Diagnose from the SURVIVING copy** (the remote / cloud
      folder), not the dead workspace: which files exist there, and
      `wc -w` each.
- [ ] Classify: **died-at-zero** (nothing usable survived — resume
      needs only the plan) vs **half-landed** (N chapters survived —
      resume needs a continuity re-read first).
- [ ] Close the dead run honestly: mark its note incomplete and add a
      **seam pointer** — exactly what was done, exactly where the
      successor picks up. Do not delete; do not restart in a fresh
      folder.

## Resuming (half-landed path)

- [ ] Re-read every survived chapter against `CANON.md` and the
      promise manifest BEFORE drafting chapter N+1.
- [ ] Note any continuity repairs needed in DECISIONS.md; make them
      as part of the resume, not "later."
- [ ] Continue on the SAME record/branch/note — append, don't fork.
- [ ] Adopt the corrective habit now, in this run (e.g. push after
      every commit), and record that you did.

## Closing the resume

- [ ] Full draft complete; band measured (`wc -w`); aimed pass done.
- [ ] The run's note flipped from in-progress to complete — as a
      deliberate last act, only when it's true.
- [ ] The death + resume recorded in DECISIONS.md in one honest
      sentence (what died, what survived, what changed).
